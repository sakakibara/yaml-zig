# Bench fixtures

- `small.yaml` (~1 KB): hand-written config-like document; also the
  target of the typed `parseInto` benchmark (its shape mirrors the
  `Config` struct in `bench/main.zig`).
- `medium.yaml` (~22 KB): a multi-document Kubernetes-style manifest
  (Deployment specs with nested metadata, label maps, container specs,
  env lists, resource requests/limits, and probes).
- `large.yaml` (~290 KB): a flat sequence of 1000 records mixing
  strings, ints, floats, bools, nulls, nested maps, and small flow
  sequences.

`medium.yaml` and `large.yaml` are generated by `generate.sh`
(deterministic; no timestamps or randomness, so reruns are
byte-identical). The generated files are checked in so `zig build bench`
needs no generation step.
